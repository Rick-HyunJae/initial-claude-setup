# FSD 레이어 구조

## 계층 순서 (상위 → 하위)

```
app       ← 가장 상위 (전역 설정, 진입점)
pages     ← 라우트 단위 화면
widgets   ← 독립적으로 재사용되는 UI 블록
features  ← 사용자 행동 단위 기능
shared    ← 가장 하위 (인프라, 공통 도구)
```

import는 반드시 **하위 레이어 방향**으로만 허용된다.

---

## 레이어별 정의

### `app/`

전역 설정, Provider, 라우터, 전역 스타일을 담는다.

| 구분     | 내용                                    |
| -------- | --------------------------------------- |
| 역할     | 애플리케이션 초기화 및 진입점           |
| 허용     | Provider 설정, 라우터 구성, 전역 스타일 |
| 금지     | 비즈니스 로직, 도메인 코드              |
| 슬라이스 | 없음 (세그먼트로만 구성)                |

```
app/
├── providers/
├── styles/
└── index.ts
```

---

### `pages/`

라우트 하나에 대응하는 화면 단위. 하위 레이어를 조합한다.

| 구분     | 내용                                                        |
| -------- | ----------------------------------------------------------- |
| 역할     | 라우트별 화면 구성                                          |
| 허용     | 해당 페이지에서만 쓰이는 UI, 상태, API 포함 가능 (FSD v2.1) |
| 금지     | 다른 page에서 직접 import                                   |
| 슬라이스 | 있음 (페이지명 기준)                                        |

```
pages/
└── profile/
    ├── ui/
    ├── model/
    ├── api/
    └── index.ts
```

---

### `widgets/`

여러 페이지에서 재사용되는 독립적인 UI 블록. 자체 상태 포함 가능.

| 구분     | 내용                                   |
| -------- | -------------------------------------- |
| 역할     | 복합 UI 블록 (헤더, 사이드바, 피드 등) |
| 허용     | 자체 상태, API 포함 가능 (FSD v2.1)    |
| 금지     | 다른 widget에서 직접 import            |
| 슬라이스 | 있음                                   |

```
widgets/
└── header/
    ├── ui/
    ├── model/
    └── index.ts
```

---

### `features/`

재사용 가능한 사용자 행동 단위. "사용자가 ~할 수 있다"로 설명된다.

| 구분     | 내용                                                      |
| -------- | --------------------------------------------------------- |
| 역할     | 독립된 사용자 시나리오 (로그인, 좋아요, 장바구니 추가 등) |
| 허용     | 비즈니스 로직, 상태, API 포함                             |
| 금지     | 다른 feature에서 직접 import                              |
| 슬라이스 | 있음                                                      |

```
features/
└── auth/
    ├── ui/
    ├── model/
    ├── api/
    └── index.ts
```

---

### `shared/`

비즈니스 도메인에 무관한 공통 인프라. 어디서든 재사용 가능.

| 구분     | 내용                                                           |
| -------- | -------------------------------------------------------------- |
| 역할     | UI Kit, 유틸리티, API 클라이언트 기반 설정                     |
| 허용     | UI 컴포넌트, 유틸 함수, API 클라이언트, 라우트 상수, 공통 타입 |
| 금지     | 비즈니스 로직, 도메인 특화 코드                                |
| 슬라이스 | 없음 (세그먼트로만 구성)                                       |

```
shared/
├── ui/
├── lib/
├── api/
├── config/
└── types/
```

---

## 레이어 선택 기준 요약

| 상황                         | 레이어      |
| ---------------------------- | ----------- |
| 앱 전역 설정, 라우터         | `app/`      |
| 특정 라우트 화면             | `pages/`    |
| 여러 페이지에서 쓰는 UI 블록 | `widgets/`  |
| 재사용되는 사용자 행동       | `features/` |
| 비즈니스 무관 공통 도구      | `shared/`   |
